# Soul Alignment

## Overview

The Soul Alignment skill is designed to ensure that the technical work being performed—whether it's refactoring code, building portfolios, or professional outreach—remains deeply connected to the user's "Life Quest." It acts as a reflective filter to prioritize actions that build both tangible value (career stability) and long-term personal authority.

## When to Use

- Before starting a new, high-stakes development task.
- When feeling disconnected from the broader career or personal goals.
- To audit completed work for its alignment with personal growth and financial stability.
- During reflective checkpoints (e.g., end of week) to assess the trajectory of the "Life Quest."

## Reflective Framework

When using this skill, analyze the current work against these three pillars:

1. **Technical Mastery:** Does this action expand technical competence in high-leverage domains (AI, Web, Distributed Systems)?
2. **Career Authority:** Does this build a visible proof-of-competence that signals "senior builder" status to recruiters?
3. **Practical Stability:** Does this work directly contribute to fast cashflow, outreach, or professional standing?

## Verification Checklist

- [ ] Does this task serve the "Life Quest" (growth/financial stability)?
- [ ] Is this the highest-leverage use of time right now?
- [ ] Does this task build a visible artifact or asset (blog post, project, profile)?
- [ ] Are technical choices aligned with modern, high-growth stacks (e.g., Bun, AI tooling)?

## One-Shot Recipes

### Task Audit
1. Pause execution.
2. Read the current TODO item.
3. Apply the pillars: "Does this advance mastery, authority, or stability?"
4. If yes, proceed. If no, re-prioritize.

### Weekly Review
1. Review the `/home/m7r/projects/moughamir.github.io/dev-log.md` file.
2. Synthesize progress in Obsidian using the "Magistus" reflective framework.
3. Adjust the next week's plan to focus on the highest-leverage "Life Quest" goals.
